# 药业智能咨询助手 - 使用说明

## 一、快速启动

### 方式1：一键启动（推荐）
双击项目根目录的 **`一键启动.bat`**
- 自动启动Flask后端服务
- 自动启动ngrok公网隧道（可选，给手机/外部访问用）
- 启动完成后自动打开浏览器

### 方式2：手动启动
```bash
# 进入项目目录
cd RAG-YIYAO

# 激活虚拟环境
.\venv\Scripts\Activate.ps1

# 启动服务
python run.py
```

首次启动前，请复制 `.env.example` 为 `.env`，填入自己的 LLM API Key。`.env` 不要提交到 Git。

### 启动后访问地址
| 入口 | 地址 | 说明 |
|------|------|------|
| 药学咨询助手 | http://localhost:5006 | 主页面，AI用药咨询 |
| RAG测试管理 | http://localhost:5006/admin/rag-logs | 查看测试记录、处理未回答问题、补知识库 |
| 库存预测看板 | http://localhost:5006/forecast | 5000商品销量预测+补货建议 |
| 公网访问 | 见启动时显示的ngrok地址 | 手机/外部访问用 |

### 关闭服务
关掉最小化的 `Flask-5006` 窗口即可（或Ctrl+C）

## 二、功能模块

### 模块1：AI药学咨询助手（http://localhost:5006）

**能做什么：**
- 药品用法用量查询（"布洛芬缓释胶囊怎么吃？"）
- 不良反应查询（"有什么副作用？"）
- 禁忌人群查询（"孕妇能吃吗？"）
- 症状咨询（"最近牙龈疼怎么办？"）
- 多轮对话追问（"那可以用什么药？"→ 自动理解上下文）
- 闲聊（"你好"→ 秒回，不调大模型）

**使用方法：**
1. 在输入框输入问题，按回车或点"发送"
2. 系统自动走10层安全链路：脱敏→分诊→检索→生成→禁用词过滤→免责声明
3. 回答流式输出，边生成边显示

**快捷测试按钮：**
页面顶部有6个快捷按钮，点击即可测试不同场景：
- 依托度酸片怎么服用？（正常药品查询）
- 布洛芬不良反应（不良反应查询）
- 描述症状咨询（症状→药品推荐）
- 急症测试（胸痛→应被拦截引导就医）
- 阿莫西林怎么吃？（超纲/混合检索测试）
- 闲聊测试（你好→本地秒回）

**多轮对话示例：**
```
用户：布洛芬缓释胶囊有什么不良反应？
AI：  布洛芬耐受性良好，不良反应一般为肠胃不适或皮疹...
用户：吃了之后会不会胃不舒服？        ← 追问，自动理解"布洛芬"
AI：  服用后可能出现肠胃不适，建议餐中或饭后服用...
用户：小孩能吃吗？                    ← 追问，自动理解"布洛芬"
AI：  ...
```

**安全机制演示：**
- 输入"突然胸痛呼吸困难" → 急症拦截，不推药，引导就医
- 输入"阿莫西林怎么吃" → 混合检索命中（关键词召回+向量检索）
- 输入"你好" → 闲聊秒回，不调大模型

### 模块1.1：RAG测试管理后台（http://localhost:5006/admin/rag-logs）

**能做什么：**
- 查看所有本地测试问题和回答记录
- 按已回答、待补知识库、需要澄清、无效输入、需复核筛选
- 给问题标记已处理、待补知识库、需复核、忽略
- 导出测试记录 CSV
- 对未回答问题填写人工审核后的标准答案和来源，保存为待入库知识
- 将审核后的知识追加到 `data/drug_knowledge.json`，并触发向量库重建

**公网测试建议：**
在 `.env` 中设置 `ADMIN_TOKEN=你自己的后台密码`。后台支持通过页面输入 token，或用 `?token=xxx` 访问。不要把真实 `.env` 提交到 Git。

**知识库更新流程：**
1. 在主页面让测试用户正常提问
2. 到测试管理后台筛选“待补知识库”
3. 填写标准问题、标准答案、来源
4. 保存待审核条目
5. 审核通过后点击“入库”
6. 点击“重建向量库”
7. 重建完成后重新测试该问题

注意：医药知识必须人工确认来源，不能把用户原话自动当成知识写入。

### 模块2：AI库存预测看板（http://localhost:5006/forecast）

**能做什么：**
- 5000商品未来30天销量预测
- 补货量建议（预测销量+安全库存）
- 数据漂移检测（消费习惯变化预警）
- AI生成可执行补货操作清单

**使用方法：**

1. **查看总览**
   - 顶部5个KPI卡片：商品总数、预测总销量、建议总库存、漂移商品数、平均MAPE

2. **筛选商品**
   - 左侧侧边栏点击品类筛选（化学药/中成药/保健食品等）
   - 搜索框输入商品名搜索
   - 下拉框切换排序（建议库存/预测销量/MAPE/漂移程度）
   - 勾选"只看漂移商品"查看需要重训的商品

3. **查看预测图表**
   - 点击表格中任意一行
   - 下方显示该商品的ECharts预测曲线：
     - 绿色实线：历史60天销量
     - 橙色虚线：未来30天预测
     - 灰色点线：测试集真实值（验证预测精度）

4. **生成AI补货建议**
   - 点击"生成建议"按钮
   - AI基于5000商品预测数据生成操作清单：
     - 整体趋势结论
     - 紧急补货TOP5
     - 漂移预警
     - 预测不准需人工复核的商品
     - 可执行操作清单

**数据说明：**
- 预测模型：ExponentialSmoothing（statsmodels）
- 商品数据：按大型连锁药房品类结构生成5000商品
- 历史数据：365天模拟销量（含季节性+趋势+星期效应+节假日）
- 评估指标：MAPE（平均绝对百分比误差），14.2%为良好水平

## 四、技术架构（10层安全链路）

```
用户提问
  ↓
第1层 脱敏过滤        → 手机号/身份证/地址替换为占位符
第2层 分诊评估        → 急症/重症拦截引导就医，轻症/中症放行
第2.4层 无效输入拦截  → 单字符/纯数字直接回复
第2.5层 闲聊拦截      → "你好/谢谢"本地模板秒回
第3层 Query Rewrite   → 多轮追问补全主语（"有什么禁忌"→"依托度酸片有什么禁忌"）
第4层 向量检索        → bge-small-zh + 关键词混合召回，从4万条知识找TOP5
第5层 超纲检测        → 相似度<0.65拦截，防止大模型编造
第6层 问题类型识别    → 4种回答风格（用法用量/不良反应/禁忌/一般）
第7层 大模型生成      → GLM API流式输出，基于知识库不编造
第8层 禁用词检测      → "根治/包治/100%有效"自动替换
第9层 免责声明        → 分级免责（药品查询/症状建议/就医引导）
第10层 审计日志       → 记录问题哈希/分诊级别/违规词
  ↓
最终回答返回用户
```

## 五、常见问题

### Q: 启动后页面打不开？
A: 等待20-30秒（embedding模型加载中），刷新页面。确认5006端口没被占用。

### Q: 回答很慢？
A: 首次提问需要加载模型约10秒，后续会快。追问会多一次Query Rewrite调用（约1秒）。如果API限流会自动切换备用API。

### Q: 库存预测看板数据为空？
A: 需要先运行预测脚本生成结果：
```bash
python "src/24_inventory_forecast_5000.py"
```
或直接运行全量预测：
```bash
python "data/forecast_5000_result.json"  # 已预生成则无需再跑
```

### Q: 急症测试不推药？
A: 这是设计的安全机制。输入"胸痛/呼吸困难/昏迷"等急症关键词会被分诊层拦截，直接引导就医，不推荐药品。

### Q: 知识库没有的药怎么说"未收录"？
A: 超纲检测层判断相似度<0.65会拦截，返回"知识库暂未收录"，不让大模型编造。

### Q: 公网地址打不开？
A: ngrok免费版地址每次启动会变，需要当天启动当天用。如果没装ngrok，只用localhost访问。
