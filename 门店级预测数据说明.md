# 门店级库存预测数据说明

本项目新增了一套门店级模拟数据，用于后续 Transformer / TFT / DeepAR 等多序列模型实验。

## 数据口径

当前旧版 9000 商品数据是聚合 SKU 口径：

```text
sku_id + date
```

新版门店级数据是：

```text
store_id + sku_id + date
```

默认规模：

```text
140 家门店 × 5000 个 SKU × 730 天 = 511,000,000 个门店-SKU-日期点
```

数据目录：

```text
data/store_sku_140x5000_2y
```

## 文件结构

| 文件 | 说明 |
|---|---|
| `product_master.csv` | 商品主数据，含具体药品名、通用名、品牌名、厂家、规格、批准文号、品类等 |
| `product_aliases.csv` | 商品别名/包装名/ERP名映射，用于模拟 SKU 归一化 |
| `store_master.csv` | 门店主数据，含门店城市、等级、商圈、医保店、大仓编号等 |
| `calendar.csv` | 日期维度，含星期、月份、节假日窗口 |
| `assortment_uint8.npy` | 门店是否经营某 SKU，shape = `(store, sku)` |
| `sales_qty_uint16.npy` | 日销售量张量，shape = `(store, sku, day)` |
| `stock_qty_uint16.npy` | 日末库存张量，shape = `(store, sku, day)` |
| `inbound_qty_uint16.npy` | 当日入库量张量，shape = `(store, sku, day)` |
| `stockout_uint8.npy` | 是否缺货，shape = `(store, sku, day)` |
| `sample_sales_long.parquet` | 小样本长表，便于查看 ERP 明细形态 |
| `dataset_summary.json` | 数据集摘要 |

## 生成脚本

```powershell
.\venv\Scripts\python.exe .\src\32_generate_store_sku_dataset.py `
  --stores 140 `
  --skus 5000 `
  --days 730 `
  --out-dir .\data\store_sku_140x5000_2y `
  --store-chunk 5 `
  --force
```

脚本支持小规模验证，例如：

```powershell
.\venv\Scripts\python.exe .\src\32_generate_store_sku_dataset.py `
  --stores 20 `
  --skus 1000 `
  --days 730 `
  --out-dir .\data\store_sku_20x1000_2y `
  --force
```

## 后续建模建议

第一阶段建议先做模型对照：

| 模型 | 粒度 | 目的 |
|---|---|---|
| SeasonalNaive / MovingAverage | 单个门店-SKU | 作为简单基线 |
| 日期回归模型 | 单个门店-SKU | 对照旧版最佳思路 |
| Transformer / TFT / DeepAR | 多门店-SKU 联合训练 | 验证跨门店、跨商品共享学习是否提升 |

训练样本建议：

```text
输入过去 90 天销售、库存、缺货、日历特征
预测未来 30 天销售
```

注意：缺货日不能直接当成真实低需求。训练时应把 `stockout_uint8.npy` 作为特征或 mask，避免模型把断货销量误学成需求下降。
