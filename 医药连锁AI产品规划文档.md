## 一、产品总览

### 1.1 产品定位

```
AI 药师助手 —— 不是AI医生

能做的（药师边界）：
  ✅ 药品用法用量咨询
  ✅ 不良反应/禁忌查询
  ✅ OTC药品推荐
  ✅ 库存预测与补货建议
  ✅ 门店运营数据分析

不能做的（医生边界）：
  ❌ 疾病诊断
  ❌ 开处方
  ❌ 处方药推荐
  ❌ 替代医生面诊
```

### 1.2 产品矩阵（7大模块）

```
                     医药连锁 AI 产品矩阵
                          │
     ┌────────┬───────────┼───────────┬──────────┐
     │        │           │           │          │
  【知识底座】 【用户触点】  【运营提效】  【风控防线】
     │        │           │           │
  ⑥知识库    ①药学咨询    ③门店运营    ④合规风控
   问答       ②会员营销    ⑤库存预测
     │        │           │           │
     └────────┴─────┬─────┴───────────┘
                   │
              ⑦流程自动化
          （贯穿所有模块的粘合剂）
```

### 1.3 落地优先级

| 批次 | 模块 | 优先级 | 理由 |
|---|---|---|---|
| 第一批 | ⑥知识库→③门店运营→①药学咨询 | ★★★ | 快赢、低风险、有demo基础 |
| 第二批 | ④合规风控→②会员营销 | ★★ | 上线门槛必须做 |
| 第三批 | ⑤库存预测→⑦流程自动化 | ★ | 高价值但技术门槛高 |

## 二、7大模块详细规划

---

### 模块①：AI 智能药学咨询

#### 业务痛点
- 执业药师不足，夜间/偏远门店无人答用药问题
- 顾客咨询排队，体验差
- 药师水平参差不齐，回答不一致

#### AI方案
| 技术 | 作用 | demo状态 |
|---|---|---|
| RAG向量检索 | 从2万条药品知识库检索用法用量 | ✅ 已跑通 |
| 微调模型（7B） | 固化药学咨询回答风格 | ✅ 已跑通 |
| 答案校验回调 | 4维度校验答案质量 | ✅ 已跑通 |
| 脱敏过滤 | 保护用户隐私 | ✅ 已跑通 |
| 分诊系统 | 判断轻重缓急决定推药还是转就医 | ✅ 已跑通 |

#### 技术架构
```
用户提问
  ↓
脱敏过滤（手机号/身份证→占位符）
  ↓
分诊评估
  ├─ 急症 → 建议就医，不推药
  ├─ 重症 → 建议就医，不推药
  ├─ 中症 → 推OTC药 + 建议就医
  └─ 轻症 → 推OTC药 + 观察
  ↓
症状分析（LLM提取可能药品）
  ↓
RAG检索（bge-small + numpy搜索2万块）
  ↓
超纲检测（相似度<0.60→兜底）
  ↓
微调模型生成（7B LoRA，4种风格不机械）
  ↓
答案校验（忠实度/相关性/安全/格式灵活度）
  ↓
合规风控（禁用词/处方药拦截/强免责）
  ↓
返回用户
```

#### 核心指标
| 指标 | 目标 | demo实测 |
|---|---|---|
| 回答准确率 | >90% | 98%（句子边界切分） |
| 风格一致性 | 100%带安全提示 | 100%（微调固化） |
| 超纲兜底率 | 100%不编造 | 100%（相似度阈值） |
| 急症拦截率 | 100% | 100%（规则引擎毫秒级） |
| 响应时间 | <30秒 | 20-30秒 |

#### 合规边界
- 只推OTC非处方药，处方药转人工
- 不做诊断，只做症状分析+用药咨询
- 每条回答必带免责声明
- 急症/重症不推药直接引导就医

### 模块②：AI 会员精准营销

#### 业务痛点
- 会员复购率低，营销一刀切
- 慢病用药周期没追踪，复购提醒不及时
- 营销内容可能违规（夸大疗效）

#### AI方案
| 技术 | 作用 |
|---|---|
| 用户画像 | 消费记录+慢病档案→标签体系 |
| 用药周期预测 | 时序预测下次复购时间 |
| 推荐算法 | 基于画像推荐相关品类 |
| 合规审核 | 营销内容过禁用词检测 |

#### 数据需求
- 会员消费记录（POS系统）
- 慢病档案（高血压/糖尿病用药周期）
- 会员基本信息（脱敏后使用）

#### 跟其他模块协同
- 跟①共用用户画像
- 跟⑤共用预测算法
- 营销内容过④合规审核

#### 优先级：★★ 中（需会员体系成熟）

#### 业务痛点
- 店长每天花2小时做报表
- 排班/补货决策靠经验
- 数据散落各系统，查询难

#### AI方案
| 技术 | 作用 | demo状态 |
|---|---|---|
| Text-to-SQL | 自然语言查销售数据 | ✅ 原RAG项目已跑通 |
| 异常检测 | 自动发现销售额/毛利率异常 | 规划中 |
| 智能日报 | 自动生成门店日报 | 规划中 |

#### 技术架构（基于已有RAG项目）
```
店长提问"上个月销售额最高的10家门店"
  ↓
Text-to-SQL（LLM生成ClickHouse SQL）
  ↓
执行SQL查真实数据
  ↓
LLM生成KPI卡片+ECharts图表+文字分析
  ↓
返回数据看板
```

#### 核心指标
| 指标 | 目标 |
|---|---|
| SQL生成准确率 | >85% |
| 查询响应时间 | <10秒 |
| 店长使用率 | >60% |

#### 优先级：★★★ 高（已有完整项目经验）

### 模块④：AI 合规风控

#### 业务痛点
- 医药强监管，AI答错一句话可能触发处罚
- 违规宣传（根治/包治/无副作用）
- 处方药违规推荐
- 超范围售药

#### AI方案：4层防护
| 层 | 防护内容 | demo状态 |
|---|---|---|
| 第1层 输入侧 | 脱敏+禁止问题拦截（诊断/药物滥用/购药引导） | ✅ 已跑通 |
| 第2层 检索侧 | 超纲检测+处方药检测 | ✅ 已跑通 |
| 第3层 生成侧 | 禁用词检测+安全提示检测+诊断语言自动替换 | ✅ 已跑通 |
| 第4层 输出侧 | 强免责+审计日志+人工兜底 | ✅ 已跑通 |

#### 禁用词清单
```
疗效夸大：根治/包治/百分百/药到病除/祖传秘方
绝对化：最好/最佳/第一/唯一/绝对/无副作用
违规承诺：无效退款/保证治好
诱导消费：限时优惠/马上抢购
非法行医：确诊/诊断为/你得了/你患了
```

#### 人工兜底触发条件
- 答案审核不通过（有违规词/缺安全提示）
- 涉及处方药
- 分诊为急症/重症
- 用户主动要求人工

#### 审计日志
- 每个问答留痕（时间/用户哈希/问题/答案/分诊/审核）
- 存储在 logs/audit_log.jsonl
- 出问题可追溯

#### 为什么不能只靠prompt
1. prompt会被绕过（用户巧妙提问可诱导违规输出）
2. prompt是软约束（LLM可能不遵守）
3. 代码层是硬约束（规则引擎100%拦截）
4. 审计日志是prompt做不到的
5. 人工兜底是prompt做不到的

#### 优先级：★★★ 高（上线门槛，不做不能上线）

### 模块⑤：智能库存预测

#### 业务痛点
- 药品有有效期，备多过期损耗，备少断货丢销售
- 千店规模靠人工订货不现实
- 季节性药品（感冒药冬天畅销）要提前备

#### AI方案
| 技术 | 作用 | demo状态 |
|---|---|---|
| Prophet时序预测 | 预测未来30天销量 | ✅ 50种药品已跑通 |
| 特征工程 | 趋势+季节性+星期+节假日 | ✅ |
| 数据漂移检测 | 消费习惯变了提醒重训 | ✅ 17种检测到漂移 |
| LLM结论生成 | 把数字转成可执行补货建议 | ✅ 已跑通 |

#### 技术架构
```
历史销量数据（365天×50种）
  ↓
Prophet本地预测（7.5秒跑完50种）
  ↓
评估MAPE（平均17.7% = 优秀）
  ↓
库存建议（预测销量+安全库存=补货量）
  ↓
数据漂移检测（偏移>20%需重训）
  ↓
LLM生成结论（紧急补货/正常/暂停/重训）
  ↓
店长执行
```

#### 核心指标
| 指标 | 目标 | demo实测 |
|---|---|---|
| MAPE | <25% | 17.7%（优秀） |
| 预测速度 | <30秒/50种 | 7.5秒 |
| 漂移检测 | 覆盖所有药品 | 17/50检测到 |
| 补货建议 | 可执行 | 4紧急/17正常/3暂停 |

#### 部署方式
- Prophet：本地CPU运行，免费，不需API
- LLM结论：调外部API（3API轮询）

#### 优先级：★★ 中（高价值但技术门槛高）

## 四、合规风控体系

### 4.1 法律法规依据
- 《药品经营质量管理规范(GSP)》
- 《互联网诊疗管理办法》
- 《药品网络销售监督管理办法》
- 《广告法》（禁用绝对化用语）

### 4.2 4层防护架构

```
第1层 输入侧
  ├─ 脱敏：手机号/身份证/地址→占位符
  ├─ 禁止问题：诊断/药物滥用/购药引导/婴幼儿用药/拒绝就医
  └─ 命中即拦截

第2层 检索侧
  ├─ 超纲检测：相似度<0.60→不展示编造答案
  └─ 处方药检测：注射液/精神类→标注+转人工

第3层 生成侧
  ├─ 禁用词：根治/包治/百分百/无副作用→替换为***
  ├─ 安全提示：必须含"咨询/医生/药师"
  ├─ 诊断语言：你得了→您可能是
  └─ 自动修复

第4层 输出侧
  ├─ 强免责声明（5种模板）
  ├─ 引导就医（分诊级别决定）
  ├─ 审计日志（每个问答留痕）
  └─ 人工兜底（审核不过/处方药/急症→转人工）
```

### 4.3 分诊系统

| 级别 | 处理方式 | 回复语气 |
|---|---|---|
| 急症 | 不推药，建议就医 | 冷静专业，不制造恐慌 |
| 重症 | 不推药，建议就医 | 建议式 |
| 中症 | 推OTC药+建议就医 | 建议+免责 |
| 轻症 | 推OTC药+观察 | 用药建议+免责 |

### 8.2 关键认知（面试必答）

1. **RAG vs 微调**：RAG给事实，微调改风格，组合用
2. **预测 vs 查询**：Prophet算未来数字，LLM给可执行结论
3. **合规不能只靠prompt**：代码层硬约束+审计+人工兜底
4. **不做诊断**：AI是药师助手不是医生，引导就医是底线
5. **数据漂移**：消费习惯变了模型要重训
6. **混合部署**：本地模型管常见问题，外部API管溢出
